## рЯОѓ CURRENT PLAN вАФ Plan 17: Active Page Context (IMPLEMENT NEXT)

### Problem
- Visible page lives only in `Workspace` local `useState(activeId)`
- `ResetGroup` falls back to `pages[0]` when no selection
- History navigate: `Workspace` effect clears `focusedRowKey` too early (row flash broken)
- `HistoryTab` in PropertiesPanel has no click-to-navigate (only restore)

### Tasks

**1. editorStore.ts**
- Add `activePageId: string | null`
- Add `setActivePageId(id: string | null)`
- `navigateTo(pageId, rowKey?)` вЖТ `setActivePageId(pageId)` + set `focusedRowKey` + auto-clear flash after 1200ms
- Do NOT clear `focusedRowKey` when consuming `navigateToPageId` in Workspace

**2. Workspace.tsx**
- Replace local `activeId` state with `activePageId` from editorStore (or sync both ways once)
- `goToPrev` / `goToNext` / PageList `onSelect` вЖТ `setActivePageId`
- `navigateToPageId` effect: only `setActivePageId` + clear `navigateToPageId` вАФ **remove `clearFocusedRow()` there**
- Keep invalid-page fallback after reflow rebuild

**3. PropertiesPanel вАФ ResetGroup**
```ts
pageId: selection?.pageId ?? activePageId ?? pages[0]?.id
```

**4. PropertiesPanel вАФ HistoryTab**
- Click entry (not restore btn) вЖТ `navigateTo(entry.pageId, entry.layerKey ?? row key)`
- Restore button: `stopPropagation`

**5. Optional:** `src/lib/editorContext.ts`
- `getContextPageId()` = selection?.pageId ?? activePageId ?? pages[0]?.id

### Verification
- Open page 2, no selection вЖТ page-scope Reset clears only page 2 overrides
- History entry click вЖТ jumps to page + row flash ~1.2s
- `npx tsc --noEmit` + `npm run build` + `verify-editor.mjs`
