You are acting as the Working Agent (Cursor/Lovable). Your task is to implement a major architectural feature: **Cross-Page/Cross-Surah Text Reflow & Dynamic Linking Interaction**.

### Feature Requirements:

**1. Dynamic Linking Application (Based on Scope & Tool):**
When the user has a specific scope selected (e.g., General, Page, Surah, Global) and they interact with ANY of the 3 text layers (Arabic, Bangla, Symbol) using either the Selection Tool (Move) or the Type Tool:
- If the **Linking Switch** is ON, the modification (movement, styling, typing) must automatically propagate and apply to all linked lines based on the selected scope. 
- Ensure that the scope selection correctly filters which lines get affected dynamically as the user works.

**2. Interactive Text Reflow (Line Breaking & Overflow):**
This is the core complex feature. The user must be able to use the Type Tool to edit text dynamically with cascading reflow:
- If the user places the cursor in the middle of a line (e.g., the last ayah of Surah Fatiha on Page 1) and presses `Enter`, the line must break, pushing the text after the cursor to the next line.
- If they press `Enter` again, it should create an empty line, pushing the subsequent text further down.
- **Cross-Page Cascading:** If the pushed text exceeds the maximum line capacity for the current page, the overflowing line(s) MUST automatically move to the **Next Page**.
- **Cross-Surah Cascading:** If the overflow reaches a Surah boundary, it must correctly affect the subsequent Surah's layout.

**3. Reflow Confirmation Dialog (CRITICAL UX):**
Because cross-page and cross-surah reflows are destructive and can alter the layout of the entire book, you MUST implement a confirmation mechanism:
- Whenever a user action (like pressing Enter or pasting text) causes a line to overflow and move to **another page** or affect **another Surah**, you must intercept this action and show a native React confirmation modal (e.g., using Radix UI `AlertDialog`).
- **Dialog Message Example:** "এই পরিবর্তনটির ফলে কিছু টেক্সট পরবর্তী পেজ বা সূরায় চলে যাচ্ছে। আপনি কি নিশ্চিত যে আপনি লেআউট পরিবর্তন করতে চান?" (This change causes some text to overflow to the next page/surah. Are you sure you want to alter the layout?)
- If the user confirms, apply the cascading reflow algorithm across the pages. If they cancel, abort the text insertion/line break.

### Implementation Guidelines:
- You will likely need to heavily modify or utilize `src/state/reflowStore.ts` and `src/state/editorStore.ts` to support this page-to-page data movement algorithm.
- Ensure the state updates correctly preserve the Symbol, Arabic, and Bangla alignment for the reflowed lines.
- Write robust logic for the cascading effect so that pushing one line to Page 2 pushes the last line of Page 2 to Page 3, and so on, recursively if needed.

Please review the current state management architecture and implement this feature comprehensively.
